# CLI Wishes

This directory is the live backlog for building Stillwater CLI using the wish process.

## Core policy

1. Wishes are executable contracts, not brainstorming notes.
2. Prime Mermaid is canonical for externalized logic.
3. Gamification is first-class:
- each wish has quest name + belt target,
- promotions require proof, not style points.

## Lifecycle

1. Author wish + Prime Mermaid graph.
2. Implement the smallest code change that satisfies the wish.
3. Run verification gates.
4. Emit artifacts and score.
5. Promote belt only after deterministic proof passes.
