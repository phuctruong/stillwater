## Key Protocols (Auto-Active)

### 1. Lane Algebra (Epistemic Typing)
Every claim automatically gets typed:
- **A-lane:** Proven facts (tests pass, mathematical proof)
- **B-lane:** Framework assumptions (well-established facts)
- **C-lane:** Heuristics (LLM confidence, pattern-based)
- **STAR:** Unknown (insufficient information)

MIN rule: `combine(lanes) = MIN(...)` — weakest premise dominates.

### 2. Counter Bypass (Hybrid Intelligence)
When counting/aggregating:
1. LLM classifies items
2. CPU enumerates (exact, deterministic)
3. Result: 99.3% accuracy (vs 40% pure LLM)

### 3. Verification Ladder (3-Rung Proofs)
All code/logic verified through:
- **641 (Edge Sanity):** Basic functionality on 10 cases
- **274177 (Stress Test):** 10,000 edge cases, adversarial inputs
- **65537 (Formal Proof):** Mathematical guarantee of correctness

Failure probability: ≤ 10^-7 (safer than human code)

### 4. Red-Green Gate (TDD Enforcement)
All code requires:
1. Failing test (RED) proving bug exists
2. Patch passes test (GREEN)
3. Proof certificate signed

No patches without RED-GREEN transition.
